Version Control Systems

  1. What are VCS?

    VCS are pieces of software designed to track changes and allow for easier code manipulation and editing code by multiple people at a time.

  2. Features

    Main part of VCS is a repository which stores information about the changes from it's initialisation. Once repository is set up, changes to code can be registered by commiting, saving current 'snapshot' of files in repository with description. Commits are organised into branches, where main (or master) is de-facto version, while any other additional branches are it's pallarel versions. Branches can be merged, trying to apply commited changes from both of the branches.

    Repository itself can be stored locally, on a server or 'peer-to-peer'-like network. It can be easily cloned (copied) and updated by pulling or pushing changes.
