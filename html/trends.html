Modern Trends and real life use

  • GitOps

    Is a practice of using git to control everything. It gives a lot of control and allows for easy restoration/rollback if soemthing goes wrong, as well as keeping track of chanegs made to configuration files.

  • Cloud based collaboration

    Uses one of the platforms like GitHub, GitLab etc. to collaborate on codebase on a company-wide scale. Not only git features are used, but additional features of platforms like issues and pull/merge requests which make working in a team much easier than using pure git or any other VCS. The procedure can vary depending of inner policies in a company.

  • AI assisted code reviews

    AI is used to help with VCS, in cases like merge conflicts and help to detect errors/ issues on if it has access to repository as context. This approach arguments and automates some part of the work.

Development pipeline

In summary real-life use case for VCS nowaday may look like this:

  1. Creating a branch for specific case/ issue
  2. Editing code on local repository copy
  3. Making a pull request for the changes
  4. Automated and manual code testing and review
  5. Either:
    1. Pull request is not accepted and it's back to step 2
    2. Pull request is accepted and merged into main branch
  6. Changes are automatically deployed
  7. If needed changes can be reverted
