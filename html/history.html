History of VCSs

Timeline

  • 1972 Source Code Control System (SCCS)

  • 1982 Revision Control System (RCS)

  • 1986 Concurrent Versions System (CVS)

  • 2000 Subversion (SVN)

  • 2005 Git and Mercurial

  • 2008 GitHub launched

  • 2010+ GitOps

Most notable versions

  1. CVS

    Used up to 2000. Allowed programmers to work on same code simultaneously. Commits weren't atomic (some files could be ommited from the changes and not tracked properly) and branches functionality wasn't the best.

  2. SVN

    Improvement to CVS. Commits were atomic and branching features were enhanced. However it was still centralised (repository on central sever)

  3. Git
